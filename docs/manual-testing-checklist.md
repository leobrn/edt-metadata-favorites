# Чек-лист ручного тестирования

Чек-лист дополняет автоматические тесты проверками в реальном 1С:EDT. Перед отправкой изменения
выполняются все применимые разделы. Перед выпуском релиза выполняется полный прогон.

## Протокол проверки

- Версия или commit:
- Версия 1С:EDT:
- Операционная система:
- Версия Java:
- Проверяющий:
- Дата:

Для пропущенного пункта указывается причина и оценка риска. Обнаруженные ошибки сопровождаются шагами
воспроизведения и релевантным фрагментом `Error Log`.

## Предусловия

- [ ] Подготовлена 1С:EDT 2026.1 без установленной проверяемой сборки.
- [ ] Подготовлен отдельный workspace минимум с двумя открытыми проектами конфигураций.
- [ ] В одном проекте присутствуют объекты с формами, реквизитами, табличными частями и командами.
- [ ] Для проверки производительности доступна конфигурация с большим числом объектов.
- [ ] Выполнена release-сборка:

  ```powershell
  .\build-release.ps1
  ```

- [ ] Maven завершился с `BUILD SUCCESS`, автоматические тесты прошли без ошибок и пропусков.
- [ ] Версия release не содержит `-SNAPSHOT`.

## Установка

- [ ] В `.build/releases/<version>/` созданы update-site ZIP, bundle JAR,
  `release-notes.md` и `SHA256SUMS.txt`.
- [ ] Контрольные суммы из `SHA256SUMS.txt` совпадают с release-артефактами.
- [ ] `release-notes.md` соответствует секции `Unreleased` в `docs/CHANGELOG.md`.
- [ ] Плагин устанавливается через `Help → Install New Software` без ошибок зависимостей.
- [ ] После перезапуска в списке установленного ПО присутствует `EDT Metadata Favorites`.
- [ ] После запуска в `Error Log` нет ошибок активации `edt.metadata.favorites`.
- [ ] Если проверяется обновление, новая сборка устанавливается поверх предыдущей публичной версии,
  а сохранённое избранное остаётся доступным.

## Навигатор и основные команды

- [ ] Контекстное меню проекта содержит команды добавления и удаления проекта из избранного.
- [ ] Контекстное меню объекта содержит команды одиночного и рекурсивного добавления и удаления.
- [ ] Недоступные для текущего состояния команды не отображаются.
- [ ] После добавления проекта или объекта появляется декоратор избранного.
- [ ] После удаления декоратор исчезает без перезапуска EDT.
- [ ] `Ctrl+Alt+K` изменяет состояние выбранного элемента Навигатора.
- [ ] Кнопка избранного в редакторе изменяет состояние открытого объекта.
- [ ] `Ctrl+Alt+K` работает в редакторе без выделения объекта в Навигаторе.
- [ ] При переключении редакторов состояние и изображение кнопки обновляются.

## Фильтр «Только избранное»

- [ ] Кнопка фильтра включает и выключает режим «Только избранное».
- [ ] Включение фильтра не отключает другие активные фильтры Навигатора.
- [ ] Избранный проект без отдельно выбранных объектов отображается целиком.
- [ ] После добавления отдельного объекта остаются объект и полный путь к нему.
- [ ] Проект, не добавленный целиком, отображается при наличии избранных объектов.
- [ ] Ранее раскрытые пользовательские узлы не сворачиваются после обновления.
- [ ] При изменении избранного через меню, горячую клавишу или диалог дерево обновляется сразу.
- [ ] После перезапуска EDT состояние фильтра и состав избранного соответствуют сохранённым данным.

## Рекурсивное избранное

- [ ] «Добавить в избранное с вложенными» добавляет текущий состав ветки.
- [ ] Новый дочерний объект внутри рекурсивно выбранной ветки наследует избранное.
- [ ] Одиночное удаление потомка создаёт исключение и сохраняется после перезапуска.
- [ ] Повторное добавление исключённого потомка возвращает его в избранное.
- [ ] Удаление вложенной ветки исключает её существующих и будущих потомков.
- [ ] Исключение вложенной ветки не влияет на соседние ветки того же рекурсивного родителя.

## Диалог управления избранным

- [ ] Диалог открывается через `Ctrl+Alt+F` и меню Навигатора.
- [ ] В списке проектов присутствуют все открытые проекты конфигураций.
- [ ] Ветка «Общие», категории и вложенные группы соответствуют структуре Навигатора.
- [ ] Одиночные и групповые отметки корректно отражают полное и частичное состояние.
- [ ] «Выбрать все» и «Снять все» изменяют только текущий проект.
- [ ] Режим «Только выбранные» скрывает пустые группы.
- [ ] Переключение проекта не теряет несохранённые изменения текущей сессии диалога.
- [ ] `ОК` применяет изменения; `Отмена` не изменяет сохранённое состояние.
- [ ] Закрытие изменённого диалога через `Escape` или крестик запрашивает подтверждение.

## Поиск в диалоге

- [ ] Один введённый символ не запускает фильтрацию.
- [ ] Начиная со второго символа поиск оставляет совпадения и полный путь к ним.
- [ ] Регистр символов не влияет на результат.
- [ ] Найденный вложенный объект можно выбрать независимо от родителя.
- [ ] Режим «Только выбранные» корректно совмещается с поиском.
- [ ] При количестве результатов до 300 пути к совпадениям раскрываются автоматически.
- [ ] При количестве результатов более 300 дерево фильтруется без массового раскрытия.
- [ ] Очистка строки возвращает полное дерево и не сбрасывает несохранённые отметки.
- [ ] На большой конфигурации ввод, очистка поиска и массовые операции не блокируют UI
  на неприемлемое время.

## Хранение и жизненный цикл объектов

- [ ] Избранное сохраняется после закрытия и повторного запуска EDT.
- [ ] Переименование объекта не сбрасывает избранное, поскольку ключом остаётся UUID.
- [ ] После переименования читаемая FQN-метка обновляется в `pinned.yaml`.
- [ ] Удалённый объект очищается из избранного после успешной загрузки конфигурации.
- [ ] Удаление вложенного объекта не удаляет существующих соседних объектов.
- [ ] Удалённый корень рекурсивной ветки исчезает из сохранённого состояния.
- [ ] Состояние разных проектов с одинаковыми именами объектов не смешивается.
- [ ] Ошибка записи в state location регистрируется в `Error Log` и не приводит к аварийному
  завершению EDT.

## Завершение проверки

- [ ] Все обязательные и применимые пункты выполнены.
- [ ] Для пропущенных пунктов приведено обоснование.
- [ ] Блокирующие и критические ошибки отсутствуют.
- [ ] Известные ограничения и результаты ручной проверки указаны в описании изменения.
