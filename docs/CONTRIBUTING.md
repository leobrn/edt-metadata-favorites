# Contributing

Перед отправкой изменений:

1. Создайте отдельную ветку от актуальной основной ветки.
2. Ограничьте изменение одной задачей и добавьте тесты для новой логики.
3. Выполните `mvn -f app/pom.xml clean verify`.
4. Пройдите все применимые пункты
   [чек-листа ручного тестирования](manual-testing-checklist.md).
5. Опишите назначение изменения, результаты автоматической и ручной проверки,
   пропущенные пункты checklist с обоснованием и известные ограничения.

Полный checklist обязателен перед выпуском релиза. Для изменений только в документации допускается
не выполнять runtime-проверки, если это явно указано в описании изменения.

Сообщение об ошибке должно содержать версию 1С:EDT, шаги воспроизведения,
ожидаемый и фактический результат, а также релевантный фрагмент журнала ошибок.
